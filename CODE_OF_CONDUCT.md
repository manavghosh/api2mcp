# Code of Conduct

## Our Pledge

API2MCP is committed to providing a welcoming, inclusive, and respectful environment
for everyone — regardless of age, background, experience level, nationality, or identity.

We as contributors and maintainers pledge to make participation in this project a
positive experience for everyone.

## Our Standards

Examples of behaviour that contributes to a positive environment:

- Using welcoming and inclusive language
- Being respectful of differing viewpoints and experiences
- Gracefully accepting constructive feedback
- Focusing on what is best for the community
- Showing empathy towards other community members

Behaviour that is not acceptable in this project is defined in the full
Contributor Covenant v2.1, which this project adopts:

> https://www.contributor-covenant.org/version/2/1/code_of_conduct/

## Enforcement

Instances of unacceptable behaviour may be reported by opening a GitHub Discussion
marked as **private**, or by emailing the project maintainer directly (see the
repository contact information).

All reports will be reviewed and investigated promptly and fairly. Maintainers are
obligated to maintain confidentiality with regard to the reporter.

## Scope

This Code of Conduct applies within all project spaces — GitHub Issues, Pull Requests,
Discussions, and any other forum where the project is represented — and also applies
when an individual is representing the project in public spaces.
