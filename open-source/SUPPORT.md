# Support

This document explains where to get help with API2MCP.

---

## Documentation

The first place to look is the official documentation:

- **Getting Started** — installation and your first MCP server
- **CLI Reference** — all commands and options
- **Tutorials** — step-by-step guides for common workflows
- **API Reference** — full module and class documentation

> Docs site: `https://manavghosh.github.io/api2mcp/`

---

## GitHub Discussions

For questions, ideas, and general discussion:

**https://github.com/manavghosh/api2mcp/discussions**

Use the appropriate category:

| Category | Use for |
|----------|---------|
| **Q&A** | "How do I…?" questions |
| **Ideas** | Feature suggestions and proposals |
| **Show and Tell** | Share what you built with API2MCP |
| **General** | Anything else |

Discussions are indexed by search engines, so your question and its answer may
help future users too.

---

## GitHub Issues

Issues are for **bugs** and **confirmed feature requests** — not general questions.

Before opening an issue:

1. Search existing issues (open and closed) to avoid duplicates
2. Check the documentation and Discussions
3. Try the latest version of API2MCP

If you have a bug, use the [Bug Report template](.github/ISSUE_TEMPLATE/bug_report.md).

---

## Security Vulnerabilities

Do **not** report security issues in public GitHub Issues.
See [SECURITY.md](SECURITY.md) for the private reporting process.

---

## Stack Overflow

Tag your questions with `api2mcp` on Stack Overflow. The maintainers monitor this tag.

---

## Response Expectations

API2MCP is maintained by volunteers. We aim to:

- Respond to GitHub Discussions within **5 business days**
- Triage new Issues within **5 business days**
- Fix confirmed bugs within **30 days** (critical: 14 days)

There is no SLA for community support. For production use cases requiring guaranteed
response times, consider engaging a commercial support provider or sponsoring the project.
