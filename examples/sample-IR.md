# Sample IR (Intermediate Representation)

Generated by parsing `tests/fixtures/petstore.yaml` (OpenAPI 3.0.3) through `OpenAPIParser`.

## What is the IR?

The IR (Intermediate Representation) is the **central data structure** in API2MCP. It decouples parsing from generation:

```
OpenAPI Spec ─┐                    ┌─> MCP Server code
GraphQL Spec ─┼─> Parser ─> IR ─> Generator ─┤
Postman Spec ─┘                    └─> LangGraph tools
```

- Every **parser** (OpenAPI, GraphQL, Postman) outputs an `APISpec` IR
- Every **generator** (MCP Tool Generator, LangGraph Adapter) consumes the same IR
- Adding a new parser or generator requires **zero changes** to the other side

## IR Dataclass Hierarchy

```
APISpec                          # Top-level container
├── title, version, description  # API metadata
├── base_url                     # Primary server URL
├── source_format                # "openapi3.0", "openapi3.1", "graphql", etc.
│
├── servers: list[ServerInfo]    # All server URLs with variables
│   └── url, description, variables
│
├── endpoints: list[Endpoint]    # Every API operation
│   ├── path, method, operation_id, summary
│   ├── parameters: list[Parameter]
│   │   └── name, location (path|query|header|cookie), schema, required
│   ├── request_body: RequestBody | None
│   │   └── content_type, schema, required
│   ├── responses: list[Response]
│   │   └── status_code, description, content_type, schema
│   ├── pagination: PaginationConfig | None   # Auto-detected
│   │   └── style (offset|cursor|page), page_param, limit_param, cursor_param
│   ├── tags, security, deprecated
│   └── metadata: dict           # Extension data
│
├── auth_schemes: list[AuthScheme]
│   ├── name, type (apiKey|http_bearer|http_basic|oauth2|openIdConnect)
│   ├── api_key_name, api_key_location    # For apiKey type
│   ├── scheme, bearer_format             # For http type
│   ├── flows                             # For oauth2 type
│   └── openid_connect_url               # For openIdConnect type
│
├── models: dict[str, ModelDef]  # Named schemas from components
│   └── name, schema: SchemaRef, description
│
└── metadata: dict               # openapi_version, tags, external_docs
```

### SchemaRef (the recursive schema type)

`SchemaRef` maps to JSON Schema and is used everywhere schemas appear (parameters, request bodies, responses, models):

```
SchemaRef
├── type          # "string", "integer", "number", "boolean", "array", "object", "null"
├── description, format, pattern, default, example
├── nullable      # true if field can be null
├── enum          # Allowed values
├── required      # Required property names (for objects)
├── properties    # dict[str, SchemaRef] — nested object fields
├── items         # SchemaRef — array item schema
├── additional_properties  # For map/dictionary types
├── one_of, any_of, all_of  # Composition schemas
├── ref_name      # Original $ref name for traceability (e.g., "Pet")
├── min_length, max_length, minimum, maximum  # Validation constraints
└── to_json_schema()  # Method to convert back to JSON Schema dict
```
